ContextSearch web-ext 1.9.0

* Attention ContextSearch web-ext LITE Users *

ContextSearch web-ext Lite has been depreciated, no longer receiving updates and will be removed from the add-on store. ContextSearch web-ext Lite users should export their settings and switch to ContextSearch web-ext 1.8.0+ to continue receiving updates for ContextSearch web-ext.

Native Messaging permissions have been moved from the standard version to a separate add-on, so there are no permissions differences between the standard v1.8.0 and the previous lite versions.

Changes for 1.9.0

* Phasing out auto importing via native app by using new browser.search API in FF 63+. OCSEs (One-Click Search Engines) are imported into CS on launch. They cannot be edited from the search engine manager, however, unless they are imported using the import search.json.mozlz4 button. OCSEs are tagged with a marker in the manager. On search.json.mozlz4 import, nodes pointing to OCSEs will be updated to reference the imported engine, making them editable.

* Search engines can be re-ordered from the Toolbar search and Quick menus via drag&drop

* Add-on size has been reduced 20% by trimming some legacy code, compressing icons, etc

* Added Remember Search History to Toolbar options.

* Hotkeys can be set for bookmarklets in Quick menu

* Sidebar with opening widget. Widget can be moved left or right, and offset relative to the top of the page via dnd. Widget responds to text dnd. (dnd not available in Chrome)

* Search engine tiles respond to dnd text (dnd not available in Chrome)

* Toolbar search tile formatting matches quick menu tile formatting. FF has a width limit to browser_action windows and so may not accommodate a high number of search engine columns

* web_accessible_resources removed from FF (needed in Chrome)

* Added option to disable Auto Quick menu when selecting more than (n) characters

* Added option to set the initial opacity of the Quick menu. Most useful for auto opening ( ~0.2 shown, default 1.0 )

* Added user-defined stylesheets

* Added option to disable menu animations

* Added lite / dark themes

Fixes

* Toolbar search hides rows if entire row is made up of hidden tools

* Toolbar close on click not working

* Using arrow keys in Quick menu with single column/labels applying class incorrectly causing large icons and missing text

* Context menu displays nodes with duplicate ids

* Toolbar shows search bar when Quick menu search bar position set to hidden

* Adding a new folder opens name for editing

* Short timeout (50ms) in auto opening to filter some accidental double-click selection events

* Overflow / scrollbar handlers for Toolbar Search / Sidebar
